# 💳 신용카드 이상거래 탐지 (Credit Card Fraud Detection)

> 머신러닝 기반 불균형 이진 분류 프로젝트  
> 전체 거래의 **0.17%** 만 사기인 극심한 불균형 데이터에서 사기 거래를 탐지합니다.

---

## 📌 프로젝트 개요

신용카드 거래 데이터에서 사기(Fraud) 거래를 정확히 탐지하는 분류 모델을 구축합니다.

**핵심 문제:**  
단순히 "전부 정상"이라고 예측해도 정확도 99.8%가 나오는 불균형 데이터  
→ Accuracy 지표는 의미가 없으며, **F1-Score · AUC-ROC · Recall** 기준으로 모델을 평가해야 합니다.

### 📐 평가 지표

| 지표 | 설명 | 중요도 |
|------|------|--------|
| **Recall (탐지율)** | 실제 사기 중 모델이 탐지한 비율 → 놓치면 금전 피해 | ★★★ 최우선 |
| **F1-Score** | Precision + Recall 조화 평균 → 불균형 데이터 종합 지표 | ★★★ |
| **AUC-ROC** | 임계값 무관 전체 판별력 → 1에 가까울수록 좋음 | ★★☆ |
| **Precision (정밀도)** | 사기 예측 중 실제 사기 비율 → 오탐 시 고객 불편 | ★☆☆ |


"무조건 정상" 전략만으로도 정확도 99.8% → **Accuracy는 이 데이터에서 의미 없는 지표**

### 사기 거래의 3가지 데이터 패턴

#### 💰 거래 금액 패턴
- 정상 거래 평균: **$88** / 사기 거래 평균: **$122**
- 사기는 소액(1~100달러) 비중이 높음 → 소액으로 먼저 테스트 후 고액 결제
- 정상 거래는 넓은 금액대에 분산되어 있음

#### ⏰ 시간대 패턴
- **새벽 0~6시**에 사기 거래가 집중
- 정상 거래는 낮 시간대에 집중
- 카드 도용 후 짧은 시간 내 연속 결제 패턴 존재

#### 📐 피처 분포 패턴
- **V14, V17, V12**가 사기 탐지 핵심 피처 (모델 학습 결과)
- 사기 거래의 V14 값은 -10 이하로 정상 범위(-3~3)를 크게 벗어남
- 정상 거래의 V 피처는 0 근방에 집중, 사기는 극단값에 위치

## 🔍 사기 패턴 심층 분석

> 사기가 0.17%밖에 없는 이유: 카드사 실시간 감지 시스템으로 대부분 차단되고,  
> 사기범도 들키지 않기 위해 정상처럼 보이는 소규모 거래를 선호하기 때문

### 4가지 대표 사기 패턴

| 패턴 | 설명 |
|------|------|
| 💸 **소액 테스트** | 카드 도용 직후 1~10달러로 사용 가능 여부 테스트 → 성공하면 고액 결제로 이어짐 |
| 🌙 **새벽 시간대 집중** | 자동화 도구를 이용한 사기가 새벽 0~6시에 집중 → 모니터링 공백을 노린 패턴 |
| ⚡ **짧은 시간 내 반복** | 수 분~수십 분 사이에 같은 카드로 여러 번 결제 → 정상 소비 행동과 다른 이상 빈도 |
| 📉 **V14 극단값** | 모델이 찾아낸 핵심 패턴 — 사기 거래의 V14 값이 정상 범위를 크게 벗어남 |

Amount($0~$25,000)와 Time(초)의 스케일이 V1~V28과 달라 모델이 큰 값에 편향될 수 있음  
→ **컬럼마다 독립된 Scaler 적용** (평균=0, 표준편차=1)

```python
scaler_amount = StandardScaler()  # Amount 전용
scaler_time   = StandardScaler()  # Time 전용 (하나의 Scaler로 순서대로 처리하면 버그 발생)
```

### 2단계 — 훈련/테스트 분리 (8:2)

```python
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, random_state=111, stratify=y  # stratify: 사기 비율 동일 유지
)
```

### 3단계 — SMOTE (불균형 해소)

| | 정상 | 사기 |
|--|------|------|
| 적용 전 | 227,451건 | 394건 (578:1) |
| 적용 후 | 227,451건 | 227,451건 (1:1) |

> ⚠️ **SMOTE는 훈련 데이터에만 적용**  
> 테스트 데이터에 적용하면 현실에 없는 가상 데이터로 성능을 평가하게 되어 결과가 왜곡됨

## 🤖 사용 모델 (총 9개)

### 기초 모델

| 모델 | 원리 | 특징 |
|------|------|------|
| **로지스틱 회귀** | 피처에 가중치를 곱해 합산 → 확률로 변환해 판정 | 빠르고 해석 쉬움. 선형 경계만 학습 가능 |
| **결정 트리** | "V14 < -5?" 같은 조건 질문을 반복해 트리 구성 | 직관적. 단독 사용 시 과적합 위험 |
| **KNN** | 새 거래와 가장 비슷한 5개 이웃의 다수결 판정 | 이번 실험 F1 1위. 데이터 클수록 느려짐 |
| **나이브 베이즈** | 피처 독립 가정 + 조건부 확률로 클래스 판정 | 매우 빠름. 피처 간 상관관계 있으면 성능 제한 |
| **SVM** | 두 클래스 사이 여백을 최대화하는 경계 탐색 | 고차원에 강함. O(n²) 복잡도로 샘플링 필요 |

### 앙상블 모델

| 모델 | 원리 | 특징 |
|------|------|------|
| **랜덤 포레스트** | 결정 트리 100개를 다른 조합으로 학습 → 다수결 | 과적합에 강함. 안정적 성능 |
| **XGBoost** | 이전 트리의 오류에 집중해 다음 트리 학습 | scale_pos_weight로 불균형 자동 보정 |
| **LightGBM** | XGBoost와 같은 방식이지만 리프 단위 성장으로 더 빠름 | 금융권 실무 표준. 속도·성능 균형 |

### 딥러닝 모델

| 모델 | 구조 | 특징 |
|------|------|------|
| **신경망 (MLP)** | 입력(30) → 은닉층 64 → 은닉층 32 → 출력 | early_stopping으로 과적합 방지 |

### 모델별 학습 전략

| 전략 | 대상 모델 | 이유 |
|------|-----------|------|
| SMOTE 증강 데이터 사용 | 로지스틱 회귀, 결정 트리, 랜덤 포레스트, XGBoost, LightGBM, 신경망 | 대용량에서도 학습 속도 문제 없음 |

## 💡 핵심 인사이트

### 1. Accuracy는 불균형 데이터에서 의미 없다
"전부 정상"으로만 예측해도 정확도 99.8%. 반드시 **F1-Score · Recall · AUC-ROC**로 평가해야 합니다.

### 2. SMOTE는 훈련 데이터에만 적용해야 한다
테스트 데이터에 적용하면 현실에 없는 가상 데이터로 평가하게 되어 성능이 왜곡됩니다.  
이는 현업에서 가장 자주 발생하는 **데이터 누수(Data Leakage)** 실수 중 하나입니다.

### 3. 사기 탐지에서는 Recall이 Precision보다 우선이다
- 사기를 놓치면(FN) → 고객 금전 피해, 카드사 손실
- 정상을 사기로 오탐하면(FP) → 고객 불편, CS 비용  

두 오류의 비용이 다릅니다. **비즈니스 맥락이 평가 지표를 결정합니다.**

### 4. 좋은 알고리즘도 튜닝 없이는 단순 모델에 질 수 있다
KNN(가장 단순한 모델)이 XGBoost·LightGBM을 F1 기준으로 이겼습니다.  
**알고리즘 선택보다 데이터 이해와 하이퍼파라미터 튜닝이 더 중요할 때가 많습니다.**

### 5. 모델마다 맞는 전략이 따로 있다
SVM은 O(n²) 속도 문제로 샘플링이 필요하고, KNN은 SMOTE 데이터보다 원본 데이터가 더 적합합니다.  
**알고리즘의 수학적 특성을 이해해야 올바른 전략을 선택할 수 있습니다.**
